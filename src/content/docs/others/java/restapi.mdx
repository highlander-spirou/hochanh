## Entry point

Mọi ứng dụng java đều yêu cầu 1 entry point là `public static void main(String[] args)` (ko có ngoại lệ).

Mọi entry point đều expect nằm trong `src/java/com.<main_package_name>.<main_class>`. 

Trong entry point, ta decorate entry point bằng `@SpringBootApplication`. This is all magic happens.

## Controller và route registration

Khi sử dụng web framework mà ko có dependency injection, ta phải ghi danh và manage toàn bộ route thủ công. Tuy nhiên, do sử dụng depdendency injection framework, ta ko cần quan tâm. 

Để thêm 1 route (nhóm route) mới, ta decorate class bằng `@RestController`. Nhớ cái entry point đc decorate bằng `@SpringBootApplication` ko? Khi instantiate, nó sẽ parse toàn bộ project, kiếm toàn bộ các class đc decorate bằng `@RestController` và tự động register nó bằng api endpoint. Khi đã quen, ta chỉ cần copy paste 1 controller boilerplate và bắt đầu viết luôn (skip configuration step - which is the most annoying step for production)

Lưu ý, khi viết controller và method của nó, ta expect brain-dead. Meaning mọi naming phải theo convention, ko đc khác biệt.
